## Findings considered and rejected

Recorded so they aren't re-audited next run:

- **Results-response re-serialization / host re-parsing** (`server.go:handleScanResults`
  rebuilds entries+raw+failures each poll; `ipOnly` re-parses per row): real redundant
  work, but this is a localhost single-user tool polling a few times/second — the cost is
  negligible. **Not worth doing.**
- **Dependency upgrades**: `quic-go v0.60.0`, `golang.org/x/crypto v0.53.0`, and Go 1.26
  are all current; no EOL/security lag. Nothing to do.
- **`gofmt -l` flags all 14 Go files**: this is purely a CRLF line-ending artifact
  (Windows checkout), not real formatting drift — folded into plan 013 (EOL normalization),
  not a standalone finding.
- **SSRF / DNS-rebinding, path traversal, CSRF, CSP, xray process leaks, worker-pool
  races**: audited and found **already well-hardened** (the `subscriptionHTTPClient`
  `Control` hook defeats rebinding TOCTOU; `handleApplyEndpoint` has correct
  `filepath.Rel`+`Base` guards; CSRF token + loopback-Host check present; `BatchProbe`
  correctly Kill/Wait's children on every path). No findings. (Plan 009 tightens one
  *residual* input-validation gap — control chars in the endpoint value — not a hardening regression.)

## What was NOT audited

- The **live network / xray runtime paths** (actual handshakes, real 204 checks, real
  QUIC egress) — not unit-testable without network + the xray sidecar; verified by reading only.
- **Race detector**: could not run locally (no cgo toolchain). Recommend the maintainer
  run `CGO_ENABLED=1 go test -race ./...` on a machine with a C compiler as a one-time check.
- **Frontend runtime behavior / visual UI** — audited `stores.js`/`app.css`/component
  source statically (app.css is 2,387 lines but not obviously bloated; no finding raised);
  no browser/E2E pass.
- **Deep Svelte 5 rune correctness** beyond the documented VirtualTable workaround.
