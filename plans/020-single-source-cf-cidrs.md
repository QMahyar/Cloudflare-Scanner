## Scope

**In scope**: one of:

- A) checked-in `frontend/src/lib/cfRanges.js` generated by a tiny Go test or
  `go run` script from `cleanip_gen.go` (document regenerate command), or
- B) move arrays to `frontend/src/lib/cfRanges.js` hand-maintained but shared
  by component only (still two sources Go+JS), or
- C) add `GET /api/cf-ranges` returning JSON from Go vars (runtime single source)

**Prefer C if tiny** (no build codegen), or **A** if you want zero new API.

**Out of scope**: fetching live lists from cloudflare.com at runtime; docs CIDR
counts (plan 014).

## Steps

### Step 1: Choose approach C (recommended for this codebase)

Add handler in `httpserver.go` / small function:

```go
func handleCFRanges(w http.ResponseWriter, r *http.Request) {
 w.Header().Set("Content-Type", "application/json")
 json.NewEncoder(w).Encode(map[string]interface{}{
  "ipv4": cfIPv4CIDRs,
  "ipv6": cfIPv6CIDRs,
 })
}
```

Register `GET /api/cf-ranges` (CSRF not required for GET).

### Step 2: FE load presets

In `IpScanner.svelte`, replace hard-coded arrays with `$state` filled onMount
via `apiJSON('/api/cf-ranges')`, with the current arrays as **fallback** if
fetch fails (so offline UI still works).

### Step 3: Test

Go: httptest GET returns 15 + 7 entries.
FE: build succeeds.

**Verify**: `go test` + `npm run build`.
