# جایگزین آی‌پی (IP Replacer)

این تب دو حالت دارد:

1. **کانفیگ پروکسی** — کانفیگ‌های VLESS/Trojan/VMess را (از لینک اشتراک یا چسباندن مستقیم) دریافت می‌کند، با اثر انگشت (نادیده گرفتن IP:port و remark) یکتا می‌کند، سپس تمام ترکیب‌ها را با لیست اندپوینت‌های تمیز می‌سازد.
2. **WireGuard / WARP** — یک اندپوینت working واپ را روی یک یا چند فایل `.conf` اعمال می‌کند و کپی‌های تغییر یافته را در پوشه‌ای که انتخاب می‌کنید می‌نویسد.

حالت پروکسی برای به‌روز کردن اشتراک قدیمی با آی‌پی‌های تازه از اسکنر آی‌پی مناسب است. حالت WireGuard مرحلهٔ apply بعد از Endpoint Scanner است.

---

## نحوه کار

```
ورودی کانفیگ‌ها (لینک یا چسباندن)
       │
       ▼
یکتا کردن (اثر انگشت IP:port و remark را نادیده می‌گیرد)
       │
       ▼
شما انتخاب می‌کنید کدام کانفیگ‌ها استفاده شوند
       │
       ▼
شما اندپوینت‌ها را می‌چسبانید (یک خط در هر)
       │
       ▼
تولید: هر کانفیگ × هر اندپوینت
به remark هر کانفیگ " @ endpoint" اضافه می‌شود
       │
       ▼
کپی همه، دانلود فایل متنی، یا نمایش QR
```

## راهنمای قدم به قدم

### قدم ۱ — انتخاب روش ورودی

بین **Subscription URL** و **Paste Configs** جابجا شوید. فقط یکی از ورودی‌ها در هر زمان قابل مشاهده است.

#### گزینه A: لینک اشتراک

1. لینک اشتراک را وارد کنید، مثلاً `https://example.com/sub?token=xxxx`
2. روی **Fetch** کلیک کنید
3. برنامه اشتراک را دانلود می‌کند، محتوای base64 را دیکد می‌کند، تمام لینک‌های `vless://`، `trojan://` و `vmess://` را استخراج می‌کند و یکتا می‌کند

#### گزینه B: چسباندن کانفیگ

1. لینک‌های اشتراک را در textarea بچسبانید، یک خط در هر (یا جدا شده با کاما، سمیکالن، پایپ، فاصله)
2. روی **Parse** کلیک کنید
3. برنامه لینک‌های معتبر `vless://`، `trojan://` و `vmess://` را استخراج می‌کند و بقیه را نادیده می‌گیرد

**مثال جداکننده‌ها** — همه اینها کار می‌کنند:

```
vless://aaa... trojan://bbb...
vless://aaa...,trojan://bbb...
vless://aaa...;trojan://bbb...
vless://aaa...|trojan://bbb...
```

### قدم ۲ — بررسی کانفیگ‌های یکتا

بعد از دریافت یا تجزیه، لیستی از قالب‌های کانفیگ یکتا می‌بینید. کانفیگ‌هایی که فقط در IP:port یا remark متفاوت هستند در یک گروه قرار می‌گیرند.

هر ورودی نشان می‌دهد:

- پروتکل (vless / trojan / vmess)
- UUID / رمز عبور
- آدرس و پورت اصلی
- SNI، security، نوع network
- Fingerprint، host، path
- Remark

کانفیگ‌هایی که می‌خواهید استفاده کنید را با چک‌باکس انتخاب کنید. از **Select All** / **Deselect All** برای جابجایی سریع استفاده کنید.

### قدم ۳ — چسباندن اندپوینت‌ها

یک `ip:port` در هر خط در textarea اندپوینت‌ها وارد کنید. اینها می‌توانند از:

- **اسکنر آی‌پی** — بعد از اسکن تمیز روی **Push to Replacer** کلیک کنید تا آی‌پی‌های working مستقیم اینجا بیایند
- خروجی نتایج اسکنر آی‌پی (Copy All / Export Configs)
- هر منبع دیگر از اندپوینت‌های پروکسی working
- آدرس‌های دستی

### قدم ۴ — تولید کانفیگ‌ها

روی **Generate Configs** کلیک کنید. برنامه تمام ترکیب‌های کانفیگ‌های انتخاب شده × اندپوینت‌ها را تولید می‌کند.

به remark هر لینک تولید شده، اندپوینت اضافه می‌شود:

```
Remark اصلی: "US Server"
Remark تولید شده: "US Server @ 162.159.90.249:443"
```

### قدم ۵ — کپی، دانلود یا QR

- **Copy All** — تمام لینک‌های تولید شده را در کلیپ‌بورد کپی می‌کند، یک خط در هر
- **Download** — به صورت فایل `.txt` ذخیره می‌کند
- **QR** — یک QR برای اولین لینک تولید شده نشان می‌دهد (اسکن با گوشی برای ایمپورت فوری)

## جزئیات یکتا کردن

کانفیگ‌ها زمانی یکسان محسوب می‌شوند (و یکتا می‌شوند) که موارد زیر یکسان داشته باشند:

- پروتکل (vless / trojan / vmess)
- UUID
- Encryption
- Security
- SNI
- Fingerprint
- Network (tcp / ws / grpc / httpupgrade / kcp)
- Host
- Path
- Packet encoding

IP:port و remark **بخشی از اثر انگشت نیستند**، بنابراین یک کانفیگ با `1.1.1.1:443` و همان کانفیگ با `2.2.2.2:8443` یک قالب محسوب می‌شوند.

## اعمال WireGuard / WARP

وقتی می‌خواهید فایل‌های `.conf` را بازنویسی کنید (نه share URL)، تب را روی حالت **WireGuard / WARP** بگذارید.

1. یک اندپوینت working (`host:port`) را بچسبانید یا از Endpoint Scanner با **Use** / push دریافت کنید.
2. روی **Choose config(s)...** کلیک کنید و یک یا چند فایل WireGuard `.conf` انتخاب کنید.
3. در صورت نیاز **Output folder path** را تنظیم کنید:
   - خالی = ذخیره کنار executable برنامه.
   - مسیر مطلق تایپ کنید، یا **Browse** بزنید تا هر پوشه‌ای روی دستگاه انتخاب شود (محدود به پوشه نصب نیست).
4. روی **Generate Configs** کلیک کنید.

برای هر فایل، برنامه خط `Endpoint` زیر `[Peer]` را پیدا می‌کند، با اندپوینت انتخاب‌شده جایگزین می‌کند، و کپی تغییر یافته را در پوشه خروجی می‌نویسد (فقط نام فایل — مسیر آپلود حذف می‌شود). یک خط وضعیت تعداد فایل‌های ذخیره‌شده را نشان می‌دهد.

## نکات

- **استفاده با اسکنر آی‌پی:** بعد از اسکن تمیز، روی **Push to Replacer** کلیک کنید — اندپوینت‌های working مستقیم به فیلد اندپوینت‌های این تب می‌روند (حالت پروکسی)
- **استفاده با Endpoint Scanner:** بعد از اسکن WARP، یک نتیجه را به این تب بفرستید و حالت WireGuard / WARP را برای بازنویسی `.conf` انتخاب کنید
- **ترکیب‌های زیاد:** اگر ۵ کانفیگ و ۱۰۰ اندپوینت داشته باشید، ۵۰۰ لینک تولید می‌شود. برنامه این را کارآمد مدیریت می‌کند اما کلیپ‌بورد شما ممکن است محدودیت داشته باشد — برای خروجی‌های خیلی بزرگ از Download استفاده کنید
- **خطاهای fetch:** لینک‌های اشتراک محدودیت بدنه ۱۰ مگابایت دارند. اگر اشتراک شما خطا می‌دهد، مطمئن شوید لینک قابل دسترسی است و پشت صفحه ورود نیست
