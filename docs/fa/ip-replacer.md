# جایگزین آی‌پی (IP Replacer)

کانفیگ‌های VLESS/Trojan را (از لینک اشتراک یا چسباندن مستقیم) دریافت می‌کند، با اثر انگشت (نادیده گرفتن IP:port و remark) یکتا می‌کند، سپس تمام ترکیب‌ها را با لیست اندپوینت‌های تمیز می‌سازد.

عالی برای به‌روز کردن یک اشتراک قدیمی با آی‌پی‌های تازه از اسکنر آی‌پی.

---

## نحوه کار

```
ورودی کانفیگ‌ها (لینک یا چسباندن)
       │
       ▼
یکتا کردن (اثر انگشت IP:port و remark را نادیده می‌گیرد)
       │
       ▼
شما انتخاب می‌کنید کدام کانفیگ‌ها استفاده شوند
       │
       ▼
شما اندپوینت‌ها را می‌چسبانید (یک خط در هر)
       │
       ▼
تولید: هر کانفیگ × هر اندپوینت
به remark هر کانفیگ " @ endpoint" اضافه می‌شود
       │
       ▼
کپی همه یا دانلود به صورت فایل متنی
```

---

## راهنمای قدم به قدم

### قدم ۱ — انتخاب روش ورودی

بین **Subscription URL** و **Paste Configs** جابجا شوید. فقط یکی از ورودی‌ها در هر زمان قابل مشاهده است.

#### گزینه A: لینک اشتراک

1. لینک اشتراک را وارد کنید، مثلاً `https://example.com/sub?token=xxxx`
2. روی **Fetch** کلیک کنید
3. برنامه اشتراک را دانلود می‌کند، محتوای base64 را دیکد می‌کند، تمام لینک‌های `vless://` و `trojan://` را استخراج می‌کند و یکتا می‌کند

#### گزینه B: چسباندن کانفیگ

1. لینک‌های اشتراک را در textarea بچسبانید، یک خط در هر (یا جدا شده با کاما، سمیکالن، پایپ، فاصله)
2. روی **Parse** کلیک کنید
3. برنامه لینک‌های معتبر `vless://` و `trojan://` را استخراج می‌کند و بقیه را نادیده می‌گیرد

**مثال جداکننده‌ها** — همه اینها کار می‌کنند:

```
vless://aaa... trojan://bbb...
vless://aaa...,trojan://bbb...
vless://aaa...;trojan://bbb...
vless://aaa...|trojan://bbb...
```

### قدم ۲ — بررسی کانفیگ‌های یکتا

بعد از دریافت یا تجزیه، لیستی از قالب‌های کانفیگ یکتا می‌بینید. کانفیگ‌هایی که فقط در IP:port یا remark متفاوت هستند در یک گروه قرار می‌گیرند.

هر ورودی نشان می‌دهد:
- پروتکل (vless / trojan)
- UUID / رمز عبور
- آدرس و پورت اصلی
- SNI، security، نوع network
- Fingerprint، host، path
- Remark

کانفیگ‌هایی که می‌خواهید استفاده کنید را با چک‌باکس انتخاب کنید. از **Select All** / **Deselect All** برای جابجایی سریع استفاده کنید.

### قدم ۳ — چسباندن اندپوینت‌ها

یک `ip:port` در هر خط در textarea اندپوینت‌ها وارد کنید. اینها می‌توانند از:

- نتایج اسکنر آی‌پی (خروجی اندپوینت‌های working)
- هر منبع دیگر از اندپوینت‌های پروکسی working
- آدرس‌های دستی

### قدم ۴ — تولید کانفیگ‌ها

روی **Generate Configs** کلیک کنید. برنامه تمام ترکیب‌های کانفیگ‌های انتخاب شده × اندپوینت‌ها را تولید می‌کند.

به remark هر لینک تولید شده، اندپوینت اضافه می‌شود:

```
Remark اصلی: "US Server"
Remark تولید شده: "US Server @ 162.159.90.249:443"
```

### قدم ۵ — کپی یا دانلود

- **Copy All** — تمام لینک‌های تولید شده را در کلیپ‌بورد کپی می‌کند، یک خط در هر
- **Download** — به صورت فایل `.txt` ذخیره می‌کند

## جزئیات یکتا کردن

کانفیگ‌ها زمانی یکسان محسوب می‌شوند (و یکتا می‌شوند) که موارد زیر یکسان داشته باشند:

- پروتکل (vless / trojan)
- UUID
- Encryption
- Security
- SNI
- Fingerprint
- Network (tcp / ws / grpc)
- Host
- Path
- Packet encoding

IP:port و remark **بخشی از اثر انگشت نیستند**، بنابراین یک کانفیگ با `1.1.1.1:443` و همان کانفیگ با `2.2.2.2:8443` یک قالب محسوب می‌شوند.

---

## نکات

- **استفاده با اسکنر آی‌پی:** اول یک اسکن آی‌پی تمیز اجرا کنید، آی‌پی‌های working را خروجی بگیرید، سپس آنها را در فیلد اندپوینت‌های جایگزین آی‌پی بچسبانید
- **ترکیب‌های زیاد:** اگر ۵ کانفیگ و ۱۰۰ اندپوینت داشته باشید، ۵۰۰ لینک تولید می‌شود. برنامه این را کارآمد مدیریت می‌کند اما کلیپ‌بورد شما ممکن است محدودیت داشته باشد — برای خروجی‌های خیلی بزرگ از Download استفاده کنید
- **تایم‌اوت دریافت:** لینک‌های اشتراک تایم‌اوت HTTP ۳۰ ثانیه دارند. اگر اشتراک شما کند است، مطمئن شوید در این بازه پاسخ می‌دهد
