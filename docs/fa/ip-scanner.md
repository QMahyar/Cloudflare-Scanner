# اسکنر آی‌پی (IP Scanner)

آی‌پی‌های پروکسی کلاودفلر را که به اتصالات TCP پاسخ می‌دهند پیدا می‌کند و در صورت تمایل آنها را با لینک VLESS شما از طریق xray-core اعتبارسنجی می‌کند. برای پیدا کردن آی‌پی‌های تمیز کلاودفلر برای v2ray، Nekobox، Sing-box و ابزارهای مشابه مفید است.

---

## نحوه کار

اسکن در دو فاز انجام می‌شود:

```
فاز ۱: پروب TCP             فاز ۲: اعتبارسنجی xray
────────────────────        ────────────────────────
تولید آی‌پی از CIDRهای  ──>  گرفتن N تا از بهترین
کلاودفلر                     نتایج فاز ۱
      │                             │
TCP Dial به هر IP:port      شروع xray با کانفیگ
با ۵۰۰ کارگر هم‌زمان        VLESS شما، اتصال از
      │                     طریق SOCKS5، ارسال
جمع‌آوری پاسخ‌های موفق،     درخواست HTTP به
مرتب‌سازی بر اساس latency   gstatic.com
                             نگه داشتن اندپوینت‌هایی
                             که HTTP 204/200 برمی‌گردانند
                             (۱۲ کارگر هم‌زمان)
```

---

## راهنمای قدم به قدم

### قدم ۱ — وارد کردن لینک VLESS (یا خالی بگذارید)

یک لینک اشتراک `vless://...` یا `trojan://...` بچسبانید. این لینک در فاز ۲ برای اعتبارسنجی اندپوینت‌ها از طریق xray-core استفاده می‌شود.

- **اگر لینک وارد کنید:** فاز ۲ بعد از اتمام فاز ۱ اجرا می‌شود. هر اندپوینت با اجرای xray-core با این کانفیگ تست می‌شود
- **اگر خالی بگذارید:** فقط فاز ۱ (پروب TCP) اجرا می‌شود. همچنین باید **1-phase mode** را فعال کنید

پورت از روی لینک VLESS برای هر دو فاز استفاده می‌شود.

### قدم ۲ — تنظیم حالت ۱ فازه

- **غیرفعال (پیش‌فرض):** اسکن کامل دو فازه. فاز ۱ آی‌پی‌های پاسخگو را پیدا می‌کند، فاز ۲ بهترین‌ها را اعتبارسنجی می‌کند
- **فعال:** فقط پروب TCP اجرا می‌شود. پورت پیش‌فرض ۴۴۳ است وقتی لینک VLESS وارد نشده

### قدم ۳ — تنظیم تعداد اسکن

| گزینه | تعداد آی‌پی |
|---|---|
| Fast | ۵۰۰ |
| Normal (پیش‌فرض) | ۱۰۰۰ |
| Deep | ۲۰۰۰ |
| Max | ۵۰۰۰ |

### قدم ۴ — تنظیم تعداد فاز ۲

تعداد اندپوینت‌هایی که از فاز ۱ انتخاب شده و از طریق xray-core اعتبارسنجی می‌شوند:

| گزینه | زمان استفاده |
|---|---|
| ۱۰ | اعتبارسنجی سریع |
| ۲۰ (پیش‌فرض) | تعادل خوب |
| ۳۰ | کامل‌تر |
| ۵۰ | جامع |

### قدم ۵ — انتخاب ورژن آی‌پی

| گزینه | توضیح |
|---|---|
| IPv4 only (پیش‌فرض) | محدوده‌های IPv4 کلاودفلر (۲۵ CIDR) |
| IPv6 only | محدوده‌های IPv6 کلاودفلر (۹۱ CIDR) |
| IPv4 + IPv6 | هر دو، ترکیبی |

### قدم ۶ — شروع اسکن تمیز

روی **Start Clean Scan** کلیک کنید.

در طول **فاز ۱**، پیشرفت زنده را می‌بینید (`Phase 1: TCP probe — X / Y`). اندپوینت‌هایی که پاسخ می‌دهند با latency خود در نتایج ظاهر می‌شوند.

در طول **فاز ۲**، `Phase 2: xray validation — X / Y` را می‌بینید. نتایج با اعتبارسنجی هر اندپوینت به‌روز می‌شوند.

در هر زمان می‌توانید **Stop** بزنید.

### قدم ۷ — دریافت نتایج

بعد از اتمام اسکن:

1. **Download VLESS Configs** — یک فایل متنی با لینک‌های VLESS برای هر آی‌پی working دریافت کنید
2. **Download IP List** — یک لیست ساده از `ip:port` دریافت کنید

خروجی VLESS تمام پارامترهای کانفیگ اصلی (SNI، path، encryption و غیره) را حفظ می‌کند — فقط IP و پورت با هر اندپوینت working جایگزین می‌شود.

---

## جزئیات تولید آی‌پی

آی‌پی‌ها از **۲۵ محدوده IPv4 CIDR** و **۹۱ محدوده IPv6 CIDR** از محدوده‌های رسمی AS13335 کلاودفلر تولید می‌شوند.

IPv4 از **انتخاب تصادفی وزن‌دار** استفاده می‌کند — محدوده‌های بزرگتر (مثلاً `/12`) به نسبت بیشتر از محدوده‌های کوچکتر (مثلاً `/24`) انتخاب می‌شوند.

IPv6 از **انتخاب تصادفی یکنواخت** از تمام ۹۱ CIDR استفاده می‌کند.

## نکات فنی

- **فاز ۱:** از `net.DialTimeout` با تایم‌اوت ۳ ثانیه، ۵۰۰ گوروتین هم‌زمان، محدود شده با semaphore
- **فاز ۲:** هر اندپوینت یک فرآیند xray مجزا با پورت SOCKS5 منحصر به فرد (شروع از ۲۰۸۰۰) می‌گیرد. ۱۲ اعتبارسنجی همزمان. تایم‌اوت ۵ ثانیه
- **اعتبارسنجی:** xray به اندپوینت متصل می‌شود، سپس برنامه یک درخواست HTTP از طریق پروکسی SOCKS5 به `www.gstatic.com/generate_204` می‌فرستد. HTTP 204 یا ۲۰۰ موفقیت محسوب می‌شود
- **TCP-only:** فاز ۱ فقط اتصال TCP را چک می‌کند. هیچ داده‌ای نمی‌فرستد — فقط اتصال برقرار و بسته می‌شود
