# اسکنر آی‌پی (IP Scanner)

آی‌پی‌های پروکسی کلاودفلر را که به اتصالات TCP پاسخ می‌دهند پیدا می‌کند و در صورت تمایل آنها را با لینک VLESS شما از طریق xray-core اعتبارسنجی می‌کند. برای پیدا کردن آی‌پی‌های تمیز کلاودفلر برای v2ray، Nekobox، Sing-box و ابزارهای مشابه مفید است.

---

## نحوه کار

اسکن در دو فاز انجام می‌شود:

```
فاز ۱: پروب TCP              فاز ۲: اعتبارسنجی xray
──────────────────────        ──────────────────────────
تولید آی‌پی از CIDRهای  ──>   گرفتن N تا از بهترین
کلاودفلر                      نتایج فاز ۱
      │                               │
TCP Dial به هر IP:port         شروع xray با کانفیگ
با تعداد کارگر هم‌زمان          VLESS شما، اتصال از
قابل تنظیم                    طریق SOCKS5، ارسال
      │                        درخواست HTTP به
جمع‌آوری پاسخ‌های موفق،        gstatic.com
مرتب‌سازی بر اساس latency،     نگه داشتن اندپوینت‌هایی
تشخیص colo                    که HTTP 204/200 برمی‌گردانند
```

---

## راهنمای قدم به قدم

### قدم ۱ — وارد کردن لینک VLESS (یا خالی بگذارید)

یک لینک اشتراک `vless://...` یا `trojan://...` بچسبانید. این لینک در فاز ۲ برای اعتبارسنجی اندپوینت‌ها از طریق xray-core استفاده می‌شود.

- **اگر لینک وارد کنید:** فاز ۲ بعد از اتمام فاز ۱ اجرا می‌شود. هر اندپوینت با اجرای xray-core با این کانفیگ تست می‌شود
- **اگر خالی بگذارید (و "Use Real Config" را غیرفعال کنید):** فقط فاز ۱ (پروب TCP) اجرا می‌شود

پورت لینک VLESS برای پیش‌فرض "Config port" در انتخاب پورت نیز استفاده می‌شود.

### منبع آی‌پی — مخزن کلاودفلر یا محدوده‌های دلخواه

به‌طور پیش‌فرض اسکنر از مخزن CIDR منتشرشدهٔ کلاودفلر نمونه‌برداری می‌کند. با تغییر گزینهٔ **منبع IP** به **محدوده‌های دلخواه** می‌توانید لیست خودتان را اسکن کنید. هر مورد را در یک خط بچسبانید (یا فایل `.txt` بارگذاری کنید)، با هر ترکیبی از:

| فرمت | مثال |
|------|------|
| CIDR | `104.16.0.0/13` |
| محدوده | `104.16.0.0-104.16.5.255` |
| محدودهٔ کوتاه | `104.16.0.0-255` (آخرین اوکتت را جایگزین می‌کند) |
| تک‌آی‌پی | `104.16.1.1` |
| توضیح | متن بعد از `#` یا `;` نادیده گرفته می‌شود |

IPv6 در همهٔ فرمت‌ها کار می‌کند (`2606:4700::/32`، `2606:4700::-ffff`). دکمه‌های پیش‌فرض محدوده‌های رایج کلاودفلر را با یک کلیک اضافه می‌کنند، از جمله **همه IPv4 کلاودفلر** / **همه IPv6 کلاودفلر** (لیست‌های کامل منتشرشده).

**انتخاب هوشمند:** اگر محدوده‌های شما بیش از تعداد عمق اسکن نباشند، همهٔ آدرس‌ها تست می‌شوند؛ ورودی‌های بزرگ‌تر به‌صورت تصادفی (وزن‌دار بر اساس اندازهٔ محدوده) تا تعداد عمق اسکن نمونه‌برداری می‌شوند. در حالت دلخواه انتخاب نسخهٔ IP نادیده گرفته می‌شود — دقیقاً همان خانواده‌هایی که می‌دهید اسکن می‌شوند.

### قدم ۲ — تنظیم تعداد اسکن

تعداد آی‌پی‌هایی که پروب می‌شوند:

| گزینه | تعداد آی‌پی | زمان استفاده |
|-------|-------------|-------------|
| Quick | ۱۰۰ | چک سریع |
| Normal (پیش‌فرض) | ۵۰۰ | تعادل خوب |
| Deep | ۱٬۰۰۰ | کامل |
| Insane | ۵٬۰۰۰ | خیلی کامل |
| Massive | ۱۰٬۰۰۰ | جامع |
| Custom | (شما وارد کنید) | هر عددی |

### قدم ۳ — تنظیم پروب‌های فاز ۱

تعداد **کارگرهای TCP همزمان** در فاز ۱:

| گزینه | کارگر همزمان |
|-------|-------------|
| ۱۰۰ | سبک |
| ۲۵۰ | متوسط |
| ۵۰۰ (پیش‌فرض) | استاندارد |
| ۱٬۰۰۰ | تهاجمی |
| ۲٬۰۰۰ | حداکثر |

مقدار بیشتر سریع‌تر تمام می‌شود اما CPU و حافظه بیشتری مصرف می‌کند.

### قدم ۴ — تنظیم تعداد و پروب‌های فاز ۲

**تعداد فاز ۲** — چند تا از بهترین نتایج فاز ۱ اعتبارسنجی شوند:

| گزینه | تعداد |
|-------|-------|
| ۱۰ | اعتبارسنجی سریع |
| ۲۰ (پیش‌فرض) | تعادل خوب |
| ۳۰ | کامل‌تر |
| ۵۰ | جامع |

**پروب‌های فاز ۲** — تعداد اعتبارسنجی‌های همزمان xray:

| گزینه | همزمان |
|-------|--------|
| ۵ | سبک |
| ۱۲ (پیش‌فرض) | استاندارد |
| ۲۵ | سریع |
| ۵۰ | تهاجمی |
| ۱۰۰ | حداکثر |

### قدم ۵ — انتخاب پورت‌های اسکن

انتخاب کنید کدام پورت‌های CDN کلاودفلر پروب شوند. از پیش‌فرض‌های سریع استفاده کنید:

| پیش‌فرض | پورت‌ها |
|---------|--------|
| 443 only | HTTPS 443 |
| HTTPS (6) | 443، 8443، 2053، 2083، 2087، 2096 |
| HTTP (7) | 80، 8080، 8880، 2052، 2082، 2086، 2095 |
| All (13) | هر ۱۳ پورت CDN کلاودفلر |
| Config port | پورت استخراج شده از لینک VLESS شما |

همچنین می‌توانید پورت‌های جداگانه را در گرید زیر پیش‌فرض‌ها چک/آنچک کنید.

### قدم ۶ — انتخاب ورژن آی‌پی

| گزینه | توضیح |
|-------|-------|
| IPv4 only (پیش‌فرض) | محدوده‌های IPv4 کلاودفلر (۱۵ CIDR) |
| IPv6 only | محدوده‌های IPv6 کلاودفلر (۷ CIDR) |
| IPv4 + IPv6 | هر دو، ترکیبی |

### قدم ۷ — فعال‌سازی اسکن مجاور (اختیاری)

**Scan nearby ranges** را فعال کنید تا در اطراف **هر** آی‌پی working فاز ۱ (همهٔ پاسخ‌دهنده‌های TCP، نه فقط سریع‌ترین‌ها) گسترش داده شود. برای هر آدرس IPv4 working، آی‌پی‌های اطراف `/24` پروب می‌شوند؛ برای IPv6 زیرشبکهٔ `/64`. مجموع پروب‌های مجاور سقف دارد (در حال حاضر ۴۰۹۶) تا با وجود پاسخ‌دهنده‌های زیاد محدود بماند. نتایج اسکن مجاور در لیست جداگانهٔ **Nearby** نمایش داده می‌شود.

### قدم ۸ — شروع اسکن تمیز

روی **Start Clean Scan** کلیک کنید.

در طول **فاز ۱**، پیشرفت زنده را می‌بینید (`Phase 1: TCP probe — X / Y`). اندپوینت‌هایی که پاسخ می‌دهند با latency و colo کلاودفلر (مثلاً `FRA`، `AMS`، `IAD`) نشان داده می‌شوند.

در طول **فاز ۲**، `Phase 2: Proxy validation — X / Y` را می‌بینید. نتایج با اعتبارسنجی هر اندپوینت به‌روز می‌شوند.

در هر زمان می‌توانید **Stop** بزنید — نتایج جزئی حفظ می‌شوند.

### قدم ۹ — دریافت خروجی یا ارسال نتایج

بعد از اتمام اسکن:

- **Copy All** — تمام `ip:port`ها را در کلیپ‌بورد کپی می‌کند
- **Copy Selected** — فقط ردیف‌های چک شده را کپی می‌کند
- **Export Configs** — لینک‌های VLESS/Trojan با اندپوینت‌های working جایگزین شده تولید می‌کند
- **Push to Replacer** — تمام اندپوینت‌های working را مستقیم به تب جایگزین آی‌پی می‌فرستد

## جزئیات تولید آی‌پی

آی‌پی‌ها از **۱۵ محدوده IPv4 CIDR** و **۷ محدوده IPv6 CIDR** از محدوده‌های رسمی AS13335 کلاودفلر تولید می‌شوند.

IPv4 از **انتخاب تصادفی وزن‌دار** استفاده می‌کند — محدوده‌های بزرگتر (مثلاً `/12`) به نسبت بیشتر از محدوده‌های کوچکتر (مثلاً `/24`) انتخاب می‌شوند تا پوشش یکنواختی در فضای آدرس کلاودفلر تضمین شود.

IPv6 از **انتخاب تصادفی یکنواخت** از تمام ۷ CIDR استفاده می‌کند.

هر آی‌پی تولید شده منحصربه‌فرد است — تکراری‌ها حذف می‌شوند.

وقتی **محدوده‌های دلخواه** انتخاب شده باشد، آی‌پی‌ها به‌جای مخزن کلاودفلر از ورودی خودتان می‌آیند: محدوده‌ها به بازه‌های آدرس شامل تبدیل می‌شوند و یا کامل شمارش می‌شوند (ورودی کوچک) یا بر اساس اندازهٔ بازه وزن‌دار نمونه‌برداری می‌شوند (ورودی بزرگ)، سپس با پورت‌های انتخاب‌شده ترکیب می‌شوند.

## نکات فنی

- **فاز ۱:** از `net.DialTimeout` با تایم‌اوت ۳ ثانیه، تعداد گوروتین همزمان قابل تنظیم (پیش‌فرض ۵۰۰)، محدود شده با semaphore
- **فاز ۲:** هر اندپوینت یک فرآیند xray مجزا با پورت SOCKS5 منحصر به فرد می‌گیرد. تعداد اعتبارسنجی همزمان قابل تنظیم (پیش‌فرض ۱۲). تایم‌اوت ۵ ثانیه
- **اعتبارسنجی:** xray به اندپوینت متصل می‌شود، سپس برنامه یک درخواست HTTP از طریق پروکسی SOCKS5 به `www.gstatic.com/generate_204` می‌فرستد. HTTP 204 یا 200 موفقیت محسوب می‌شود
- **تشخیص colo:** بعد از فاز ۱، هر آی‌پی پاسخگو در `/cdn-cgi/trace` پروب می‌شود تا دیتاسنتر کلاودفلر شناسایی شود
- **TCP-only:** فاز ۱ فقط اتصال TCP را چک می‌کند. هیچ داده‌ای نمی‌فرستد — فقط اتصال برقرار و بسته می‌شود
